# REST Principles Assignment

## Overview

This repository contains the solution for the REST Principles assignment. The purpose of this task is to build a strong conceptual understanding of REST (Representational State Transfer) before starting API development and implementation.

The assignment covers the fundamental concepts of REST architecture, including resources, endpoints, HTTP verbs, statelessness, and idempotency. Understanding these principles helps developers design scalable, maintainable, and reliable web APIs.

---

## Objective

Build a solid conceptual foundation in REST architecture so that the team has a shared vocabulary before touching any code.

The assignment explains:

* What a resource is and how it differs from an endpoint.
* The semantics of HTTP methods such as GET, POST, PUT, PATCH, and DELETE.
* The concepts of safe and idempotent operations.
* Why REST APIs should be stateless.
* Practical examples of idempotency and request retries.

---

## Learning Outcomes

After completing this assignment, the reader should be able to:

* Understand the core principles of REST architecture.
* Identify resources and endpoints in an API.
* Choose the correct HTTP method for different operations.
* Explain why stateless APIs are easier to scale.
* Understand the importance of idempotency in reliable systems.
